# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in Racksmith, please report it responsibly. **Do not open a public issue.**

Instead, use one of these methods:

- **GitHub Security Advisories**: [Report a vulnerability](https://github.com/ekkuleivonen/racksmith/security/advisories/new) (preferred)
- **Email**: [ekku@racksmith.io](mailto:ekku@racksmith.io)

Please include:

- A description of the vulnerability
- Steps to reproduce the issue
- The potential impact
- Any suggested fix (if you have one)

## What to expect

- Acknowledgment within **48 hours**
- A fix or mitigation plan within **7 days** for critical issues
- Credit in the release notes (unless you prefer to stay anonymous)

## Supported versions

Security fixes are applied to the latest release only.

| Version | Supported |
|---------|-----------|
| latest  | Yes       |
| older   | No        |
